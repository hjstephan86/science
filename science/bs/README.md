# Betriebssystem-Initialisierung und Echtzeit-Scheduling mit EDF+

## Hardware-Grundlagen, Boot-Sequenz, PYMCA-8-Mehrkernarchitektur und formale Korrektheitsnachweise

Diese Arbeit präsentiert ein umfassendes Konzept für ein neues Betriebssystem, das auf dem EDF+-Scheduling-Algorithmus von Stephan Epp als zentralem Paradigma aufbaut. Ausgehend von den physikalischen Grundlagen digitaler Hardware -- R/L/C-Schaltkreisen, Analog-Digital-Wandlern und Signaltheorie -- über die vollständige x86-64-Boot-Sequenz (Reset-Vektor, BIOS/UEFI-POST, ACPI, Bootloader-Kernel-Übergang) bis hin zur Prozessor-Initialisierung in Real-Modus, Protected-Modus und Long-Modus werden alle wesentlichen Schichten
eines modernen Betriebssystems formal behandelt.

Der EDF+-Scheduler erweitert klassisches Earliest-Deadline-First durch einen dynamischen Penalty-Mechanismus für Blocking-Ereignisse und wird als optimaler Algorithmus für die Hardware-Initialisierungssequenz
eingesetzt. Darauf aufbauend beschreibt die Arbeit die PYMCA-8-Mehrkernarchitektur von Stephan Epp mit acht Kernen,
Python-Speichermodell, IoFET-basiertem Dynamic-Voltage-and-Frequency-Scaling (DVFS) sowie das stochastische Archiv-Problem von Stephan Epp.
